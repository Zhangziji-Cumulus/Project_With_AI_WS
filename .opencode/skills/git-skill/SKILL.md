# Git版本控制技能

所有Git命令在项目根目录 `Project_With_AI_WS/` 下执行。默认分支为 `main`。

## 1. 提交代码

当用户说"提交代码"时，按以下步骤执行：

1. **询问用户**本次的提交信息（commit message）
2. **询问用户**本次更新类型：
   - **小版本更新** — 版本号 +0.01
   - **大版本更新** — 版本号 +0.1
   - **超大型版本更新** — 版本号 +1.0
3. **读取当前版本号**：读取 `.version` 文件内容（若不存在则默认为 `0.0`）
4. **计算新版本号**：将当前版本号解析为 `X.Y` 格式（整数部分为X，小数部分为Y），按类型对对应位增量，然后拼接为 `X.Y` 格式
5. **执行Git操作**：
   ```
   git add .
   git commit -m "用户提供的提交信息"
   git push
   ```
6. **创建版本文件**：创建 `Project_With_AI_WSversion{X.Y}.txt`，内容格式如下：
   ```
   版本号：{X.Y}
   提交日期：{yyyy-MM-dd HH:mm:ss}
   提交信息：{用户提供的提交信息}
   更新类型：{小版本更新 / 大版本更新 / 超大型版本更新}
   Commit Hash：{前7位commit hash}
   ```
7. **更新 `.version`**：将新版本号覆写回 `.version` 文件

## 2. 回退代码 / 更换代码

当用户说"回退代码"或"更换代码"时，按以下步骤执行：

1. 执行 `git pull --all` 获取远程所有引用
2. 执行 `git log --oneline -20` 显示最近20条提交记录给用户
3. 请用户选择要回退到的目标版本（通过commit hash）
4. 执行 `git checkout <commit-hash>` 或 `git reset --hard <commit-hash>` 回退到指定版本
5. 询问用户是否需要强制推送到远程（若需要，执行 `git push --force-with-lease origin main`）
6. 根据回退到的版本更新 `.version` 文件

## 3. 拉取最新代码 / 获取最新代码

当用户说"拉取最新代码"或"获取最新代码"时：

1. 执行 `git pull origin main`
2. 告知用户拉取完成
